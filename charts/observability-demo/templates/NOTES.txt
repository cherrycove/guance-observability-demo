{{- if eq .Values.gateway.service.type "LoadBalancer" }}
The Gateway is being exposed through a Kubernetes LoadBalancer.

Wait for the external address:

  kubectl -n {{ .Release.Namespace }} get svc {{ include "observability-demo.fullname" . }}-gateway-service --watch

Then visit the EXTERNAL-IP hostname over HTTP. MySQL, Redis and the internal Java services remain private.
{{- else }}
The demo is internal-only. Open it with:

  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "observability-demo.fullname" . }}-gateway-service 8080:80

Then visit http://127.0.0.1:8080/ . MySQL and Redis use emptyDir and are intentionally ephemeral.
{{- end }}

Helm generated the MySQL credentials in Secret
{{ include "observability-demo.secretName" . }}. Existing values are preserved on upgrade.
{{- else }}
The demo is using the pre-created Secret {{ include "observability-demo.secretName" . }}.
{{- end }}

Fault injection, recovery, and warm-up operations do not require a demo credential.
